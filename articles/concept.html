              # 前端面试概念

              # 原型及原型链

              • JavaScript 是基于对象的面向对象语言，而非基于类的面向对象语言。

              • 原型是ECMAScript实现继承过程中出现的一个概念，原型指在继承过程中作为基础的对象。

              • 原型链是通过生成的__proto__指针实现的。JavaScript中所有的对象都是从它的原型对象继承而来。而原型对象自身也是一个对象，它也有自己的原型对象，这样层层上溯，就形成了一个类似链表的结构，这就是原型链

              # prototype属性

              • 构造函数均有一个prototype属性，值为一个对象。包含所有实例对象需要共享的属性和方法。

              # XSS && CSRF

              • XSS，即 Cross Site Script(跨站脚本攻击)

              • XSS 攻击是指攻击者在网站上注入恶意的客户端代码，通过恶意脚本对客户端网页进行篡改，从而在用户浏览网页时，对用户浏览器进行控制或者获取用户隐私数据的一种攻击方式。

              • 目的：将一些隐私数据像 cookie、session 发送给攻击者，将受害者重定向到一个由攻击者控制的网站，在受害者的机器上进行一些恶意操作。

              • 类型：反射型（非持久型）、存储型（持久型）、基于DOM。

              • XSS攻击的防范：HttpOnly 防止劫取 Cookie（设为HttpOnly的Cookie，不允许javascript访问）、输入检查（对于用户的任何输入要进行检查、过滤和转义）、输出检查（使用编码或转义的方式来防御 XSS 攻击）。

              • CSRF，即 Cross Site Request Forgery(跨站请求伪造)

              • CSRF 攻击是攻击者借助受害者的 Cookie 骗取服务器的信任，可以在受害者毫不知情的情况下以受害者名义伪造请求发送给受攻击服务器，从而在并未授权的情况下执行在权限保护之下的操作。

              • CSRF攻击的防范：验证码、Referer Check、添加token验证

              # Vue && React的区别

              • Vue双向数据绑定、React单向
              • 模版语法不同：Vue使用扩展的HTML语法进行渲染，React使用JSX(扩展的javascript)渲染模版
              • 组件通信方法不一致：Vue：(props, $emit) provide/inject(跨越多个层级) React：(props, callback) context
              • 均有使用Vitural DOM，但略有不同
              • 监听数据变化的原理不同：Vue通过对传入vue实例的data进行遍历，使用Object.defineProperty将属性转化为getter/setter 方法，当属性发生变化时，触发setter方法，发布消息，通知订阅者，改变视图。 React通过比较引用的方式进行数据监听

              # 执行栈

              首次运行JS代码时，会创建一个globalContext并Push到当前的执行栈中。每当发生函数调用，引擎都会为该函数创建一个新的函数执行上下文并Push到当前执行栈的栈顶。
