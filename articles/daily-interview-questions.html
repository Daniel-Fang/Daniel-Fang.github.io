                                        # 杂谈---作用域与闭包

                                        • 执行环境（execution context)： 执行环境定义了变量或函数有权访问的其他数据，决定了他们各自的行为。每个执行环境都有一个变量对象VO（variable object）,执行环境中定义的所有变量和对象都保存在这个对象中。

                                          • 全局执行环境：根据宿主环境不同，全局执行环境也会不同。浏览器中，全局执行环境为 window, node 中执行环境为 global。
                                          • 函数执行环境：每个函数都有自己的执行环境，当执行流进入一个函数时，该函数的执行环境就会被 push 进一个执行环境栈中。函数执行完成后，栈将该环境弹出，将控制权交回之前的执行环境。

                                        • 作用域：作用域就是一套规则，用于确定在何处以及如何查找变量（标识符）的规则，即查找变量的地方。包括全局作用域、函数作用域和块级作用域（ES6）。

                                        • 词法作用域：也称静态作用域，由代码的书写顺序决定。

                                        • 作用域链：当在一个环境中执行时，会创建变量对象的一个作用域链（scope chain）,用途：保证对执行环境有权访问的所有变量和函数的有序访问。作用域的前端，为当前执行环境的变量对象，若执行环境为函数，将其活动对象AO (activation object) 作为变量对象。活动对象开始只包括 arguments 对象。

                                        • 闭包：闭包就是指有权访问另一个函数作用域中的变量的函数。当函数可以记住并访问所在的词法作用域时，就产生了闭包，即使函数是在当前词法作用域之外执行

                                          闭包的作用：定义模块、避免变量污染。

                                          闭包的问题：易造成内存泄漏。

                                        # day7

                                        1. iframe 框架有哪些优缺点

                                          1. 优点
                                            • 重载页面时不需要重载整个页面，只需要重载页面中的一个框架页
                                            • 技术易于掌握，使用方便，可主要应用于不需搜索引擎来搜索的页面
                                            • 可以实现跨域
                                          2. 缺点
                                            • window.onload 事件会在所有 iframe 加载完成后才触发，因此会造成页面阻塞
                                            • 对搜索引擎不友好
                                            • 多框架的页面会增加服务器的http请求

                                        2. 简述你对 BFC 规范的理解

                                          1. 特性

                                            • 计算BFC 的高度时，浮动子元素也参与计算
                                            • BFC 就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然
                                            • BFC 的区域不会与 float 的元素区域重叠

                                          2. 形成条件

                                            • html 根元素
                                            • float的值不是 none
                                            • position 的值不是 static 或者 relative
                                            • display 的值是 inline-block, table-cell, table-caption, flex, inline-flex
                                            • overflow 的值不是 visible

                                          3. 应用场景

                                            • 解决浮动子元素导致父元素，高度坍塌的问题
                                            • 解决文字环绕在float 四周的情况
                                            • 解决边距重叠问题 （父子，兄弟，空元素等）

                                        2. 对比下px、em、rem有什么不同？
                                          • px:绝对固定的值，无论页面放大或者缩小都不会改变。
                                          • em:相对父元素字体大小的倍数。如果父元素的字体为 12px，那么子元素 1em 就是 12px。由于是相对父级的倍数，所以多层嵌套时，倍数关系的计算会很头痛。
                                          • rem:相对r为root素字体大小的倍数，即相对于 html 的字体大小，如果不做任何修改，浏览器默认字体大小为 16px。

                                        # 杂谈---数据存储

                                        1. Cookie

                                          Cookie 在性质上是绑定在特定域名下的，当设置了一个 Cookie 后，给创建 Cookie 的域名发送请求的时候，都会包含这个 Cookie。

                                          1.1 Cookie 的构成

                                          • name 不区分大小写，但最好统一大小写
                                          • value 值必须被 URL 编码
                                          • 域 Cookie 对哪个域有效，所有向该域发送的请求，会自动包含 Cookie
                                          • 路径
                                          • 失效时间 expires / Max-Age 值为 GMT 格式的日期
                                          • 安全标志 Secure，为 true 时，只有使用 SSL 连接的时候发送 Cookie

                                          1.2 Cookie 使用接口并不友好，详细了解 红宝书^630^，主要使用 document.cookie 访问和设置 cookie

                                          1.3 应用场景

                                          • 保存用户登录状态，设置过期时间
                                          • 跟踪用户行为 ，比如换肤、天气地点选择

                                          1.4 限制

                                          • 每个域的 Cookie 总数一般不超过50 个
                                          • 大小不超过 4KB
                                          • 同源

                                        2. Storage

                                          2.1 常用方法

                                          • setItem(name, value)
                                          • getItem(name)
                                          • removeItem(name)
                                          • clear() 清除所有值，firefox暂不支持
                                          • key(index) 获取index 处的值的名字

                                          Storage 类型只存储字符串，非字符串在存储之前会被转换为字符串。Storage 限制因浏览器而异， 一般同源 storage 不超过 5MB。

                                          2.2 sessionStorage

                                          sessionStorage 是 Storage 的一个实例，存储特定于某个页面会话的数据。页面会话在浏览器打开期间一直保持，并且重新加载或恢复页面（即浏览器崩溃并重启，IE不支持）仍会保持原来的页面会话。在新标签或窗口打开一个页面时会在顶级浏览上下文中初始化一个新的会话。

                                          2.3 localStorage

                                          localStorage 是Storage的一个实例，要访问同一个 localStorage 对象必须同源，无过期时间，只有JavaScript 删除或是用户清除浏览器缓存。

                                          2.4 Cookie、sessionStorage、localStorage 总结

                                          2.4.1 共同点

                                          • 保留在浏览器端
                                          • 同源

                                          2.4.2 不同点

                                          • Cookie 始终会在同源的 http 请求中携带，即在浏览器、服务端来回传递，而 sessionStorage、localStorage 不会。
                                          • Cookie 有路径概念，只有指定路径，才会发送 Cookie, 后两者无此限定
                                          • Cookie 4KB, 后两者大约 5MB
                                          • 生命周期 Cookie 默认浏览器结束，删除所有Cookie, 若有设置过期时间，则过期时间之前一直有效，即便浏览器关闭。sessionStorage 一个页面回话中有效，即便刷新页面。localStorage 一直有效，除非JavaScript 删除或清除浏览器缓存。

                                        3. 应用缓存

                                          使用 manifest file，列出要下载和缓存的文件

                                        4. IndexedDB

                                          是浏览器保存结构化数据的一种数据库。是一种类似 SQL 数据库的数据存储机制，但它的数据不存在表中，而是对象中。

                                        1. 说说你对css盒子模型的理解

                                        2. var、let、const 区别
                                          • var
                                            • var 变量的作用域在其声明位置上下文中，而非声明变量总是全局的。
                                            • 声明变量是它所在上下文环境的不可配置属性，非声明变量是可配置的（如非声明变量可以被删除）。
                                            • 并且可以重复声明，会保留第一次声明而忽略后面的声明。
                                            • 存在变量提升。
                                          • let
                                            • let 声明的变量只在其声明的块或子块中可用。
                                            • let 不会在全局对象里新建一个属性。
                                            • 不可以重复声明，重复声明会报 SyntaxError
                                            • 存在变量提升，但与 var不一样
                                            • 存在暂时性死区，即未定义前不能使用。typeof 也会报错 ReferenceError
                                          • const
                                            • 和let 很类似，声明一个常量。表示变量保存的地址不变，而非值不变。
                                            • 存在暂时性死区
